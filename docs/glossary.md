# Glossary – The Silence Protocol

A reference of key terms used throughout the protocol documentation. These definitions are normative.

| Term | Definition |
|---|---|
| **Async-First** | A communication norm where written, asynchronous messages are the default; synchronous contact (calls, video) is the exception. |
| **Dark Pattern** | A design mechanism that manipulates user psychology to induce unintended behavior. Prohibited in any implementation of this protocol. |
| **Digital Peace of Mind (DPoM)** | A state in which a person controls the flow of incoming information, understands clearly what requires attention, trusts that their unavailability is respected, and has regained the capacity for sustained, uninterrupted thought. |
| **Emergency Passphrase** | A pre-agreed word or phrase that, when included in a message or call request, bypasses silence mode and delivers an immediate alert. |
| **Focus Block** | A protected period of uninterrupted work, expressed as the `deep_focus` mode. |
| **Interruption Request** | A structured, justified request from a sender to initiate synchronous contact. Must include a reason and urgency level. |
| **Inverse Presence** | Declaring the specific conditions under which one may be reached, rather than broadcasting availability. |
| **Reachability Profile** | A conditional rule set that defines how and when a user can be interrupted. |
| **Response Window** | A scheduled time range during which a user actively processes messages. Outside these windows, messages are queued without push notifications. |
| **Silence Status Object** | The core data structure representing a user's current availability state. |
| **Transparent Unavailability** | The practice of communicating unavailability clearly and unambiguously, including the expected return time, rather than remaining silent. |
